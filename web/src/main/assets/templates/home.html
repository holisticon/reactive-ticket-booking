An showcase

ReactiveTicketBooking is an online ticketing demo application that demonstrates how you can transform a traditional EE-application into a reactive akka based application with a wink. It is based on the jboss-developer Ticket-Monster showcase.

Reactive

ReactiveTicketBookings's architecture uses a subset of the rich feature set offered by Akka including streams, persistence, cluster sharding and distributed data.

Cloud-ready

A deployment of ReactiveTicketBooking runs continuously in the cloud on the OpenShift platform. Learn how easy it is to write Java EE applications that can deploy both locally and in the cloud.

Fork it on GitHub

TicketMonster is available on GitHub. Within a click, you can get the source code, build it and run it!

Learn with it

ReactiveTicketBooking is accompanied by a tutorial that walks your through the application architecture. Read it to learn how the application is built. Or even build it yourself following our instructions.

Customize it

ReactiveTicketBooking an example that you can follow, but you can also use the predefined application structure to implement your own use cases.
